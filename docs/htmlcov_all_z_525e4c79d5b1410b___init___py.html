1""" 

2Unified validation system for the framework. 

3 

4This module provides a comprehensive validation system that handles both 

5data validation and pipeline validation with early error detection and 

6clear validation messages. 

7 

8Key Features: 

9- **String Rules Conversion**: Convert human-readable rules to PySpark expressions 

10- **Early Validation**: Validate pipeline steps during construction 

11- **Data Quality Assessment**: Comprehensive data quality validation 

12- **Clear Error Messages**: Detailed validation errors with suggestions 

13- **Configurable Thresholds**: Customizable validation thresholds per layer 

14 

15String Rules Support: 

16 - "not_null" → F.col("column").isNotNull() 

17 - "gt", value → F.col("column") > value 

18 - "lt", value → F.col("column") < value 

19 - "eq", value → F.col("column") == value 

20 - "in", [values] → F.col("column").isin(values) 

21 - "between", min, max → F.col("column").between(min, max) 

22 

23Example: 

24 >>> from the framework.validation import _convert_rules_to_expressions 

25 >>> from pyspark.sql import functions as F 

26 >>> 

27 >>> # Convert string rules to PySpark expressions 

28 >>> rules = {"user_id": ["not_null"], "age": ["gt", 0], "status": ["in", ["active", "inactive"]]} 

29 >>> converted = _convert_rules_to_expressions(rules) 

30 >>> # Result: {"user_id": [F.col("user_id").isNotNull()], "age": [F.col("age") > 0], ...}
